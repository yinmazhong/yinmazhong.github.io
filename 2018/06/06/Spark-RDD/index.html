RDD

从数据层次上看：RDD是spark的灵魂

Spark在数据层面提供的主要抽象就是RDD(弹性分布式数据集)，分布在集群不同节点的不同分区中的元素的集合，
可以在这个数据集上使用各种并行算法

RDD的生成方式有两种：

  • hadoop file system,或者是hadoop-supported file system
  • Driver 程序中的scala 集合，比如Array

RDD除了是对分布式数据集的抽象外，还包含了该RDD的由来，即血缘关系图，当出现错误时，RDD可以血缘关系图恢复

两个共享变量：

  • broadcast variables
  • accumulators
